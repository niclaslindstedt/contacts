# Favorites

Star the people you reach for most and they gather on their own **Favorites**
page — the button sits next to List in the sidebar's action grid. Toggle a
favorite anywhere it shows: the **heart** in a contact card's header, or the
heart on any List or Favorites row.

Unlike the folder-grouped List, Favorites is a single **hand-orderable
shortlist** — one flat list you arrange yourself. Each row carries a **grip
handle**: drag a contact up or down to set the order, a thin insertion line
showing where it will land (on a touch screen, press and hold the grip to pick a
row up first). A newly starred contact joins the bottom until you drag it into
place.

## One number per favorite

When a contact has more than one phone number, you can mark one as their
**primary** — the number you actually reach them on. Open the contact, tap the
pencil to edit, and a **star** appears on each phone row: tap the star beside a
number to make it primary (tap the filled star again to clear the choice). The
contact's read view then tags that number with a small **Primary** star so you
can see the pick at a glance.

The Favorites page shows only the primary number for each starred contact, so a
favorite reads as a single tap-to-call rather than a stack of numbers. A favorite
that has no primary marked keeps showing its usual list.

The favorite flag and its order are stored per contact, so they ride along in the
JSON backup and sync across your devices. The same Select / Copy / Export tools
from the [List page](feature:list) are here too, and a card opened from Favorites
gets a back button to it. Every row also carries the side menu's row gestures:
**swipe left** to bare a **Delete** button, **swipe right** to archive, and
**right-click** for the **Move to folder / Archive / Delete** menu.
